# GitHub Upload Checklist

Use this if you want to upload the repository from the GitHub website without using Terminal.

## 1. Download and unzip

Download the prepared ZIP file and extract it on your computer.

You should see:

- `README.md`
- `docs/`
- `prompts/`
- `kb/`
- `evals/`
- `security/`
- `.env.example`
- `.gitignore`

## 2. Create a new repository

Recommended settings:

- **Repository name:** `queerimam-ai`
- **Description:** `Public architecture, trust design, and product method behind queerimam.ai`
- **Visibility:** Public

Do **not** pre-add a README or `.gitignore` when creating the repo, because those files are already included.

## 3. Upload the extracted contents

Inside the new repository:

- click **Add file**
- click **Upload files**
- drag in the extracted files and folders
- commit directly to `main`

## 4. Check the root looks clean

Your repository homepage should show:

- `README.md`
- `docs/`
- `prompts/`
- `kb/`
- `evals/`
- `security/`
- `.env.example`
- `.gitignore`

Do not upload the ZIP itself as the main repo content.

## 5. Polish the About section

After upload, fill in the right-hand **About** panel:

### Description
Public architecture, trust design, and product method behind queerimam.ai

### Website
Add your live queerimam.ai URL

### Topics
Suggested topics:

- ai
- llm
- rag

- flutterflow
- trust-and-safety
- islam
- lgbtq
- product-design
- safety
- knowledge-base

## 6. Format the repo for credibility

A good public repo should be easy to scan.

Make sure:

- the README opens with a strong first paragraph
- the most important claims are under clear headings
- there are links between files
- long walls of text are broken into short sections
- lists are used sparingly and only when helpful
- code or config appears inside fenced blocks where relevant

## 7. Final safety check

Before sharing the repo link, confirm that the repository does **not** contain:

- real API keys
- service-account JSON
- real vector store IDs
- user data
- internal logs
- production-only backend rules
